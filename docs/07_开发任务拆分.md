# CodeCoach AI 开发任务拆分

## 1. 文档说明

本文档用于拆分 CodeCoach AI MVP 版本的开发任务。

MVP 版本核心目标是完成“项目拷打训练闭环”：

用户注册登录 -> 创建项目档案 -> 发起项目拷打训练 -> AI 生成面试追问 -> 用户回答 -> AI 反馈追问 -> 结束训练 -> 生成训练报告。

本文档会将开发任务拆分为后端任务、前端任务、联调任务、测试任务和部署任务。

---

## 2. 任务编号规则

任务编号规则如下：

| 前缀 | 说明 |
|---|---|
| ENV | 环境与工程准备 |
| BE | 后端开发任务 |
| FE | 前端开发任务 |
| AI | AI 能力开发任务 |
| TEST | 测试任务 |
| DEPLOY | 部署任务 |
| DOC | 文档任务 |

示例：

```text
BE-001 初始化 Spring Boot 后端项目
FE-001 初始化 React 前端项目
AI-001 封装 AI 调用接口
```

---

## 3. 开发阶段总览

MVP 开发分为以下阶段：

| 阶段 | 目标 |
|---|---|
| Phase 0 | 工程初始化 |
| Phase 1 | 后端基础设施 |
| Phase 2 | 用户认证模块 |
| Phase 3 | 项目档案模块 |
| Phase 4 | AI 项目拷打核心链路 |
| Phase 5 | 训练报告模块 |
| Phase 6 | 前端页面开发 |
| Phase 7 | 前后端联调 |
| Phase 8 | 测试与修复 |
| Phase 9 | 部署上线 |

# 14. Git 提交建议

建议每完成一个小任务提交一次。

提交格式：

```text
type: description
```

常见 type：

| type | 说明 |
|---|---|
| docs | 文档 |
| feat | 新功能 |
| fix | 修复 |
| refactor | 重构 |
| chore | 工程杂项 |
| test | 测试 |
| style | 格式调整 |

示例：

```bash
git commit -m "feat: add user register api"
git commit -m "feat: add project create api"
git commit -m "docs: add development task breakdown"
```

# 15. MVP 完成标准

当以下任务完成后，MVP 第一版完成：

1. 用户可以注册并登录；
2. 用户可以创建项目档案；
3. 用户可以基于项目发起训练；
4. 系统可以生成第一道问题；
5. 用户可以提交回答；
6. 系统可以生成反馈和追问；
7. 用户可以结束训练；
8. 系统可以生成报告；
9. 前端可以完整跑通以上流程；
10. 项目可以部署到服务器。
