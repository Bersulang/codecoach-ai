# CodeCoach AI 项目立项说明

## 1. 项目名称

CodeCoach AI：程序员 AI 面试教练平台

## 2. 项目背景

当前很多 Java 后端实习求职者在准备面试时，存在以下问题：

1. 有简历和项目，但不知道面试官会如何追问；
2. 背了八股文，但不会结合项目进行表达；
3. 看了大量面经，但缺少针对自己简历和项目的模拟训练；
4. 缺少一个可以持续训练、记录反馈、定位薄弱点的工具。

CodeCoach AI 希望通过大模型能力，围绕用户输入的项目经历，生成针对性的面试追问，
并根据用户回答进行继续追问和反馈，帮助用户提升项目表达和面试应答能力。

## 3. 目标用户

第一版目标用户为：

准备 Java 后端实习 / 校招的学生。

## 4. 第一版核心目标

第一版先完成“项目拷打训练闭环”：

用户录入项目描述 -> AI 生成面试追问 -> 用户回答 -> AI 继续追问 -> 生成训练反馈。

## 5. MVP 功能范围

第一版包含以下功能：

1. 用户注册与登录；
2. 用户创建项目档案；
3. 用户输入项目名称、项目描述、技术栈、负责模块、项目亮点；
4. 系统根据项目内容生成面试追问；
5. 用户回答问题；
6. 系统根据用户回答进行追问；
7. 用户结束训练后，系统生成训练反馈；
8. 系统保存历史训练记录。

## 6. 第一版暂不包含的功能

第一版暂不包含：

1. 文档上传与解析；
2. 完整 RAG 知识库；
3. 多 Agent 协作；
4. 本地大模型部署；
5. 语音 / 视频面试；
6. 支付系统；
7. 企业组织管理；
8. 复杂管理员后台。

## 7. 第一版核心页面

1. 登录 / 注册页；
2. 项目列表页；
3. 项目创建 / 编辑页；
4. 项目拷打对话页；
5. 训练反馈页。

## 8. 第一版验收标准

当以下流程可以完整跑通，则 MVP 第一版完成：

1. 用户可以注册并登录；
2. 用户可以创建一个项目档案；
3. 用户可以基于该项目发起一次项目拷打训练；
4. AI 可以生成第一轮项目追问；
5. 用户回答后，AI 可以继续追问；
6. 用户结束训练后，系统可以生成反馈报告；
7. 用户可以查看历史训练记录。

## 9. 项目后续扩展方向

MVP 完成后，逐步扩展：

1. 简历上传与解析；
2. Markdown / PDF 项目文档上传；
3. RAG 知识库问答；
4. 八股文专项训练；
5. 真实面经题库；
6. 复习计划生成；
7. SSE 流式输出；
8. 项目公网部署；
9. 系统监控与日志分析。
