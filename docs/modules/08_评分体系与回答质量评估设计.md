# 评分体系与回答质量评估设计

## 1. 文档说明

本文档用于设计 CodeCoach AI 的评分体系与回答质量评估规则。

当前系统已经具备：

```text
项目拷打训练
八股问答训练
RAG 检索增强
用户文档 RAG
简历驱动训练
token 级流式输出
训练报告
能力画像
智能学习推荐
```

但当前评分体验存在一个关键问题：

```text
用户只回答一轮，甚至回答“不知道”，系统仍可能给出 10～40 分。
```

这会降低用户对产品的信任。

评分体系必须从“AI 生成一个看起来完整的报告”升级为：

```text
基于回答质量、轮次完成度、知识覆盖度、表达结构和追问表现的可解释评估体系。
```

---

## 2. 评分体系目标

新的评分体系要满足：

1. 分数可信；
2. 低质量回答必须低分；
3. “不知道 / 不会 / 没学过”不能获得不合理基础分；
4. 训练轮次太少时，报告必须明确样本不足；
5. 评分维度清晰；
6. 扣分原因可解释；
7. 项目训练和八股训练分别有适配 Rubric；
8. 能力画像沉淀的数据更可靠；
9. 后续训练计划和推荐能基于可信分数。

---

## 3. 当前问题

当前评分可能存在以下问题：

```text
AI 倾向于鼓励用户
AI 为了生成完整报告会给保底分
没有先判断回答是否有效
没有区分“不会”和“回答不完整”
没有根据训练轮次控制报告可信度
没有低质量回答硬性上限
没有将有效回答数量纳入评分
```

典型不合理情况：

```text
用户回答：不知道
系统评分：30 分
```

这会让用户觉得：

```text
我什么都不会也能有分
评分不可信
报告不值得看
能力画像也不可信
```

---

## 4. 核心原则

评分体系遵循以下原则：

```text
先判定回答质量，再给分。
先判断训练样本是否足够，再生成结论。
低质量回答设置分数上限。
不知道就是不知道，不能被包装成低水平理解。
```

也就是说，评分不应该只问：

```text
这次训练能给多少分？
```

而应该先问：

```text
用户是否真的回答了？
回答是否有效？
回答是否覆盖核心点？
回答是否能经得起追问？
训练样本是否足够？
```

---

## 5. 回答质量等级

所有用户回答先进行质量分级。

建议定义：

```text
NO_ANSWER
INVALID
VERY_WEAK
PARTIAL
BASIC
GOOD
EXCELLENT
```

---

## 6. NO_ANSWER

含义：

```text
用户没有有效回答。
```

典型回答：

```text
不知道
不会
没学过
不了解
不清楚
不知道怎么说
忘了
不会答
```

评分规则：

```text
单轮 NO_ANSWER 得分应接近 0。
整场训练大部分都是 NO_ANSWER，总分应在 0～15。
```

说明：

```text
可以给少量参与分，但不能给知识分。
```

---

## 7. INVALID

含义：

```text
回答无关、乱写、复制无意义内容，或明显没有回答问题。
```

典型情况：

```text
答非所问
输入乱码
只写“666”
只写“好的”
只写无关内容
```

评分规则：

```text
单轮 INVALID 得分 0～5。
整场训练大部分 INVALID，总分应在 0～10。
```

---

## 8. VERY_WEAK

含义：

```text
用户尝试回答，但只有极少信息，无法说明核心概念。
```

示例：

问题：

```text
什么是缓存击穿？
```

回答：

```text
就是缓存有问题。
```

评分规则：

```text
单轮 VERY_WEAK 得分 5～20。
整场训练大部分 VERY_WEAK，总分不应超过 25。
```

---

## 9. PARTIAL

含义：

```text
回答提到部分关键词，但缺少核心机制、适用场景或完整解释。
```

示例：

```text
缓存击穿就是热点 key 过期，请求打到数据库，可以加锁。
```

问题：

```text
没有解释为什么会发生、锁的副作用、逻辑过期等方案。
```

评分规则：

```text
单轮 PARTIAL 得分 20～45。
```

---

## 10. BASIC

含义：

```text
回答覆盖基本概念和主要方案，但表达一般，缺少追问深度。
```

评分规则：

```text
单轮 BASIC 得分 45～65。
```

---

## 11. GOOD

含义：

```text
回答准确，结构清晰，覆盖核心点，并能说明适用场景。
```

评分规则：

```text
单轮 GOOD 得分 65～85。
```

---

## 12. EXCELLENT

含义：

```text
回答完整、准确、有项目表达、有权衡、有异常场景，能经得起追问。
```

评分规则：

```text
单轮 EXCELLENT 得分 85～100。
```

---

## 13. 低质量回答硬性上限

为了避免虚高，需要设置硬性上限。

建议规则：

```text
如果回答质量为 NO_ANSWER，总分上限 15。
如果回答质量为 INVALID，总分上限 10。
如果回答质量为 VERY_WEAK，总分上限 25。
如果有效回答轮数不足 2，总分上限 50。
如果训练完成轮数不足 50%，总分上限 60。
如果没有任何有效回答，总分上限 10。
```

注意：

```text
这些上限应在最终总分阶段生效。
AI 不能绕过这些规则。
```

---

## 14. 有效回答定义

有效回答是指：

```text
不是 NO_ANSWER
不是 INVALID
不是纯寒暄
不是明显答非所问
长度和内容足以表达至少一个有效知识点
```

有效回答不等于高质量回答。

例如：

```text
提到一个关键词但解释不清
```

可以算有效回答，但质量可能只是 VERY_WEAK 或 PARTIAL。

---

## 15. 训练样本充分性

报告需要判断本次训练样本是否足够。

建议定义：

```text
INSUFFICIENT
LIMITED
ENOUGH
```

---

## 16. INSUFFICIENT

条件示例：

```text
有效回答数 = 0
或总回答轮数 = 1 且质量很低
```

报告结论：

```text
本次训练样本不足，无法形成可靠能力判断。
```

评分：

```text
总分应很低，通常 0～20。
```

---

## 17. LIMITED

条件示例：

```text
有效回答数为 1～2
或训练未完成
```

报告结论：

```text
本次训练只能反映局部表现，建议完成更多轮次后再查看能力趋势。
```

评分：

```text
总分可以给，但必须受上限约束。
```

---

## 18. ENOUGH

条件示例：

```text
有效回答数 >= 3
且训练完成度较高
```

报告结论：

```text
可以形成相对可靠的能力判断。
```

---

## 19. 八股训练评分维度

八股训练建议维度：

```text
概念准确性
核心原理理解
解决方案完整度
场景适配能力
追问应对能力
表达结构化
```

---

## 20. 八股训练维度说明

### 概念准确性

是否能准确解释概念。

例如：

```text
缓存击穿、缓存穿透、缓存雪崩是否区分清楚。
```

### 核心原理理解

是否理解底层机制。

例如：

```text
为什么热点 key 过期会导致数据库压力瞬间升高。
```

### 解决方案完整度

是否能列出并解释常见方案。

例如：

```text
互斥锁、逻辑过期、热点 key 预热、永不过期。
```

### 场景适配能力

是否知道不同方案适合什么场景。

### 追问应对能力

是否能回答更深入的问题。

### 表达结构化

回答是否有层次，而不是碎片化关键词。

---

## 21. 项目训练评分维度

项目训练建议维度：

```text
项目背景清晰度
个人贡献可信度
技术细节深度
工程权衡能力
异常场景分析
性能与稳定性意识
表达可信度
```

---

## 22. 项目训练维度说明

### 项目背景清晰度

是否能讲清项目解决什么问题。

### 个人贡献可信度

是否能讲清自己做了什么，而不是团队做了什么。

### 技术细节深度

是否能解释关键技术实现。

### 工程权衡能力

是否能说明为什么这样设计，而不是只说用了什么技术。

### 异常场景分析

是否能处理失败、降级、超时、幂等等问题。

### 性能与稳定性意识

是否能说明性能瓶颈、监控、优化。

### 表达可信度

回答是否像真实做过项目，而不是背模板。

---

## 23. 总分计算建议

总分不应该完全由 AI 自由给出。

建议：

```text
AI 负责评估维度和理由
系统负责应用上限和修正规则
```

流程：

```text
AI 输出原始评分
系统识别回答质量
系统统计有效回答数量
系统计算训练完成度
系统应用低质量上限
系统得到最终分
```

---

## 24. 分数区间解释

建议展示给用户：

```text
0～20：基本无法评价或回答无效
20～40：基础薄弱，需要先补概念
40～60：有初步理解，但回答不完整
60～75：基本合格，但深度和表达需提升
75～90：表现较好，能应对常规面试
90～100：表达成熟，能应对深入追问
```

---

## 25. 低分体验设计

低分不能只打击用户。

当用户回答“不知道”时，报告应明确：

```text
本轮回答没有提供有效信息，因此无法判断你对该知识点的掌握情况。
建议先学习基础概念，再进行训练。
```

而不是：

```text
你有一定理解，但还需加强。
```

必须避免“虚假鼓励”。

---

## 26. 报告文案原则

报告应遵循：

```text
真实
具体
可行动
不羞辱用户
不虚假鼓励
不制造焦虑
```

对于低质量回答，应说：

```text
你当前还没有展示出对该知识点的有效理解。
```

而不是：

```text
你已经掌握了一些基础。
```

---

## 27. 对能力画像的影响

能力画像只能沉淀可信数据。

建议：

```text
如果 sampleQuality = INSUFFICIENT，不应强烈影响长期能力画像。
如果回答全是 NO_ANSWER，可以记录训练失败，但不要当成稳定能力趋势。
如果训练样本有限，能力快照应标记低置信度。
```

后续可以增加：

```text
confidence
sampleQuality
validAnswerCount
```

用于能力画像聚合。

---

## 28. 前端展示要求

报告页和训练结束页应展示：

```text
最终分数
分数解释
样本充分性
有效回答数
主要扣分原因
下一步建议
```

如果样本不足，必须显著提示：

```text
本次训练样本不足，分数仅供参考。
```

如果用户回答“不知道”，应提示：

```text
当前回答没有展示有效知识点，建议先学习该知识点后再训练。
```

---

## 29. AI Prompt 要求

AI 评分 Prompt 必须明确：

```text
不要为了鼓励用户而给保底分。
如果用户回答“不知道”“不会”“没学过”，应判定为 NO_ANSWER。
低质量回答必须低分。
训练样本不足时应明确说明。
评分必须基于用户实际回答内容。
不能假设用户掌握了没有说出的内容。
```

---

## 30. 输出结构建议

AI 输出建议包含：

```json
{
  "rawScore": 38,
  "answerQuality": "PARTIAL",
  "sampleQuality": "LIMITED",
  "validAnswerCount": 1,
  "totalAnswerCount": 2,
  "dimensionScores": [
    {
      "dimension": "概念准确性",
      "score": 35,
      "reason": "用户提到了关键词，但没有解释核心机制。"
    }
  ],
  "scoreReasons": [
    "回答覆盖了部分关键词，但没有展开原理。",
    "训练轮次较少，样本有限。"
  ],
  "majorProblems": [
    "没有解释适用场景",
    "没有回答追问中的异常情况"
  ],
  "nextActions": [
    "先学习基础概念",
    "重新进行一次专项训练"
  ]
}
```

系统最终应用上限后得到：

```text
finalScore
```

## 31. 验收案例

必须通过以下案例。

### 案例一：回答“不知道”

输入：

```text
不知道
```

期望：

```text
answerQuality = NO_ANSWER
finalScore <= 15
报告说明无法判断掌握情况
建议先学习基础概念
```

---

### 案例二：答非所问

期望：

```text
answerQuality = INVALID
finalScore <= 10
```

---

### 案例三：只答关键词

输入：

```text
缓存击穿就是热点 key 过期，加锁解决。
```

期望：

```text
answerQuality = PARTIAL 或 VERY_WEAK
finalScore 不应超过 45
```

---

### 案例四：回答完整但训练轮次少

期望：

```text
如果只回答一轮，即使回答不错，总分也应受样本数量限制。
可以给较高单轮质量，但整体报告要提示样本有限。
```

---

### 案例五：完整训练且回答良好

期望：

```text
finalScore 可以进入 70～90
报告给出具体优势和不足
```

---

## 32. 总结

评分体系是 AI 面试训练产品可信度的核心。

新的评分体系必须做到：

```text
低质量回答低分
样本不足明确提示
评分理由可解释
扣分原因具体
建议可行动
能力画像数据更可靠
```

目标不是让用户舒服，而是让用户相信：

```text
CodeCoach AI 的评分是认真评估出来的。
```
