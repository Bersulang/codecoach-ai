# 产品内 AI 训练向导设计

## 1. 文档说明

本文档用于设计 CodeCoach AI 的产品内 AI 训练向导模块。

当前 CodeCoach AI 已经完成：

```text
项目拷打训练
八股问答训练
知识学习模块
能力画像模块
RAG 基础能力
知识文章 RAG
项目档案 RAG
用户文档 RAG
简历驱动训练
面试复盘 Agent
token 级流式训练体验
用户中心
我的文档
成长洞察
工作台
```

随着功能逐渐增多，用户可能会遇到一个新问题：

```text
我知道这个产品很强，但我不知道下一步该点哪里。
```

因此需要一个常驻在产品右下角的小助手。

它不是普通客服，也不是万能聊天机器人，而是一个：

```text
产品内 AI 训练向导
```

它的核心目标是：

```text
帮助用户理解当前页面、找到下一步训练动作，并快速跳转到最合适的功能入口。
```

---

## 2. 模块定位

产品内 AI 训练向导的定位是：

```text
面向 CodeCoach AI 内部功能使用场景的智能导航、训练建议与快捷操作入口。
```

它服务于：

```text
页面导航
功能解释
训练路径引导
能力画像建议
简历风险引导
复盘行动跳转
用户文档引导
学习与训练入口推荐
```

它不是：

```text
通用客服机器人
万能聊天助手
搜索引擎
简历代写助手
闲聊机器人
外部联网问答工具
自动执行高风险操作的 Agent
```

它应该像一个产品内教练：

```text
知道你在哪个页面
知道你已经完成了什么
知道你下一步可能该做什么
能给出可点击的行动按钮
```

---

## 3. 为什么需要产品内 AI 训练向导

CodeCoach AI 当前功能已经比较丰富：

```text
项目档案
项目拷打
八股问答
知识学习
成长洞察
我的文档
简历训练
复盘 Agent
训练历史
报告详情
```

功能丰富带来价值，但也带来使用门槛。

用户可能会问：

```text
我第一次进来应该先做什么？
我已经上传了简历，下一步呢？
我有成长洞察了，但该怎么练？
我在简历分析页看到风险点，应该去哪里训练？
我想练 Redis，要从哪里开始？
我想看最近问题总结，应该点哪里？
```

产品内 AI 训练向导的价值是：

```text
把分散的功能入口串成一个连续的训练路径。
```

---

## 4. 核心产品目标

第一版目标：

```text
降低用户使用门槛
提升功能发现能力
帮助用户从任何页面找到下一步行动
用 action card 连接学习、训练、复盘和简历模块
体现 AI 产品的智能引导感
```

用户打开小助手后，应该能完成：

```text
问当前页面是干什么的
问下一步该做什么
快速进入项目拷打
快速进入八股训练
快速进入简历训练
快速进入复盘 Agent
快速进入成长洞察
根据薄弱点跳转学习文章
根据简历风险进入项目训练
```

## 5. 产品形态

## 5.1 默认状态

默认状态是右下角悬浮图标。

建议表现：

```text
圆形悬浮按钮
轻微阴影
图标使用 sparkles / compass / bot / guide
固定在页面右下角
不遮挡主要操作按钮
移动端适配
```

图标可以表示：

```text
AI Guide
训练向导
面试教练
```

---

## 5.2 展开状态

点击悬浮按钮后展开小窗口。

小窗口结构：

```text
顶部区域：
- 名称：CodeCoach Guide
- 副标题：帮你找到下一步训练动作
- 收起按钮

中间区域：
- 对话消息
- 操作建议卡片
- 快捷问题

底部区域：
- 输入框
- 发送按钮
```

---

## 5.3 窗口尺寸

桌面端建议：

```text
宽度：360px ~ 420px
高度：520px ~ 640px
右下角固定
大圆角
暖白背景
轻边框
柔和阴影
```

移动端建议：

```text
接近底部抽屉
宽度接近屏幕宽度
高度不超过 80vh
```

## 7. 页面上下文解释

小助手需要知道用户当前所在页面。

例如：

```text
currentPath = /documents
```

它应该能解释：

```text
这是“我的文档”页面，你可以上传简历、README 或项目文档，让 AI 在训练时参考你的真实材料。
```

如果用户问：

```text
这个页面是干什么的？
```

助手应该根据当前路径回答，而不是泛泛解释产品。

---

## 8. 下一步训练建议

当用户问：

```text
我下一步该做什么？
```

助手应结合用户状态给建议。

可能依据：

```text
是否登录
是否有项目档案
是否上传文档
是否有简历分析
是否有训练报告
能力画像薄弱点
最近复盘结果
当前页面
```

示例：

```text
你已经上传了简历，但还没有生成简历分析。建议先进入简历训练，找出最容易被追问的项目风险点。
```

---

## 9. 快捷导航

小助手应能识别常见意图并返回可点击 action。

用户输入：

```text
我想练项目
```

返回：

```text
你可以先选择一个项目档案，然后进入项目拷打训练。
```

Action：

```text
去项目档案
```

用户输入：

```text
我想练 Redis
```

返回：

```text
你可以先查看 Redis 相关知识卡片，再进入八股专项训练。
```

Action：

```text
开始八股训练
查看知识学习
```

---

## 10. 个性化建议

如果用户已登录，助手可以读取用户数据摘要。

可以使用：

```text
能力画像摘要
最近训练报告摘要
最新复盘结果
简历风险点
智能学习推荐
项目数量
文档数量
简历数量
```

示例：

```text
你最近的薄弱点集中在 Redis 和项目表达。建议先复习 Redis 缓存击穿，再做一次项目拷打，把技术方案讲成项目经验。
```

---

## 11. 行动卡片

小助手不应该只输出文字，而应该输出行动卡片。

行动卡片字段建议：

```text
label
description
type
targetPath
priority
reason
```

示例：

```json
{
  "label": "开始 Redis 八股训练",
  "description": "围绕你最近薄弱的 Redis 方向进行专项训练。",
  "type": "TRAIN_QUESTION",
  "targetPath": "/questions",
  "priority": "HIGH",
  "reason": "你的能力画像中 Redis 相关 weaknessTags 出现频率较高。"
}
```

## 12. 支持的 action 类型

第一版建议支持白名单 action。

```text
GO_DASHBOARD
GO_PROJECTS
GO_QUESTIONS
GO_LEARN
GO_INSIGHTS
GO_DOCUMENTS
GO_RESUMES
GO_AGENT_REVIEW
GO_HISTORY
GO_PROFILE
START_PROJECT_TRAINING
START_QUESTION_TRAINING
VIEW_LEARNING_ARTICLE
VIEW_REPORT
UPLOAD_DOCUMENT
ANALYZE_RESUME
GENERATE_REVIEW
```

注意：

```text
action 必须由系统白名单控制。
不要让 AI 随意生成任意 URL。
```

---

## 13. 为什么 action 必须白名单

如果让 AI 直接返回任意路径，可能出现：

```text
返回不存在页面
跳转错误路径
暴露内部接口
生成危险链接
用户体验不稳定
```

正确做法：

```text
AI 可以表达意图
系统将意图映射为允许的 action
前端只执行白名单 action
```

## 14. 未登录状态

未登录时，小助手也可以显示，但能力有限。

未登录可做：

```text
解释产品功能
引导登录
引导查看首页
说明训练流程
```

未登录不做：

```text
读取训练记录
读取能力画像
读取简历
读取文档
生成个性化建议
```

未登录示例：

```text
登录后我可以根据你的训练报告、简历风险点和能力画像，推荐下一步训练动作。
```

Action：

```text
登录
开始体验
```

---

## 15. 已登录状态

已登录后，小助手可以更个性化。

可读取：

```text
项目数量
文档数量
简历分析状态
最近训练报告
能力画像摘要
最新复盘
智能学习推荐
```

但注意：

```text
只读取摘要，不读取完整隐私文档
不要把完整简历直接塞进 prompt
不要打印完整用户数据
```

## 16. 推荐场景

## 16.1 新用户首次进入

条件：

```text
已登录
没有项目
没有文档
没有训练记录
```

助手建议：

```text
你可以先创建一个项目档案，或者上传简历，让我帮你找到最容易被追问的项目点。
```

Actions：

```text
创建项目档案
上传简历 / 文档
```

---

## 16.2 有项目但没有训练

条件：

```text
有项目
没有项目训练记录
```

助手建议：

```text
你已经有项目档案了，可以先做一次项目拷打，看看项目亮点是否经得起追问。
```

Action：

```text
开始项目拷打
```

---

## 16.3 有文档但没有简历分析

条件：

```text
有 PDF / Markdown 文档
没有简历分析
```

助手建议：

```text
你已经上传了文档，可以选择其中一份作为简历进行分析，找出面试官可能深挖的项目点。
```

Action：

```text
去简历训练
```

---

## 16.4 有能力画像

条件：

```text
存在 ability snapshot
```

助手建议：

```text
你最近在 Redis 和项目表达上得分偏低，建议先学习 Redis 缓存击穿，再做一次专项八股训练。
```

Actions：

```text
查看成长洞察
开始八股训练
```

---

## 16.5 有复盘结果

条件：

```text
存在最新复盘 Agent 结果
```

助手建议：

```text
你最近的复盘建议优先补“逻辑过期”和“项目表达可信度”。要不要直接进入对应训练？
```

Actions：

```text
查看复盘
开始训练
```

## 17. 小助手和复盘 Agent 的区别

复盘 Agent 是一个页面级功能。

它回答：

```text
最近一段时间训练表现如何？
为什么反复出错？
下一步怎么做？
```

产品内 AI 训练向导是一个全局悬浮助手。

它回答：

```text
当前我该点哪里？
这个页面怎么用？
我现在下一步该做什么？
```

复盘 Agent 偏深度分析。

训练向导偏即时引导和导航。

---

## 18. 小助手和智能学习推荐的区别

智能学习推荐主要推荐知识文章。

训练向导不仅推荐学习，还能推荐：

```text
项目拷打
八股训练
简历训练
文档上传
复盘 Agent
成长洞察
训练历史
```

它是更高层的产品入口。

---

## 19. 是否需要真正 Agent 框架

第一版不需要复杂 Agent 框架。

推荐方式：

```text
规则引导
+ 用户数据摘要
+ 可选 AI 生成自然语言
+ action 白名单
```

不要一开始做：

```text
多步工具链
复杂规划器
长期记忆
自动执行
```

因为小助手的首要目标是：

```text
稳定、可控、好用。
```

## 20. 后端接口设计

第一版建议一个核心接口：

```http
POST /api/guide/chat
```

请求：

```json
{
  "message": "我下一步该做什么？",
  "currentPath": "/insights"
}
```

响应：

```json
{
  "answer": "你最近 Redis 相关薄弱点较多，建议先查看成长洞察中的推荐内容，再进行一次 Redis 专项训练。",
  "actions": [
    {
      "label": "查看成长洞察",
      "description": "查看你的低分维度和智能推荐。",
      "type": "GO_INSIGHTS",
      "targetPath": "/insights",
      "priority": "HIGH"
    },
    {
      "label": "开始八股训练",
      "description": "进入八股问答，选择 Redis 方向专项练习。",
      "type": "START_QUESTION_TRAINING",
      "targetPath": "/questions",
      "priority": "MEDIUM"
    }
  ]
}
```


## 22. 后端上下文聚合

后端应构造轻量上下文摘要。

建议包括：

```text
当前路径
是否登录
项目数量
文档数量
简历数量
最近训练次数
能力画像低分维度
最新复盘行动
简历高风险点摘要
智能学习推荐摘要
```

不要包含：

```text
完整简历
完整文档
完整训练回答
完整 prompt
完整向量
```

## 23. AI Prompt 设计原则

如果使用 AI 生成 answer，Prompt 必须要求：

```text
你是 CodeCoach AI 的产品内训练向导。
你的任务是帮助用户理解当前页面并找到下一步训练动作。
不要闲聊。
不要回答和产品无关的问题。
不要承诺不存在的功能。
不要生成任意链接。
只基于提供的可用功能和用户摘要给建议。
输出简洁、具体、可行动。
```

输出应为 JSON：

```json
{
  "answer": "...",
  "actionIntents": [
    {
      "type": "GO_INSIGHTS",
      "reason": "..."
    }
  ]
}
```

后端再把 `actionIntents` 映射成白名单 action。

## 24. 规则优先策略

部分问题不需要 AI。

例如：

```text
怎么上传文档？
怎么分析简历？
怎么开始项目拷打？
这个页面是干什么的？
```

这些可以规则回答，稳定性更高。

AI 更适合：

```text
我下一步该做什么？
我最近应该练什么？
我不知道从哪里开始
```

推荐第一版采用：

```text
规则优先
AI 辅助个性化建议
白名单 action 输出
```

## 26. 前端交互流程

用户流程：

```text
进入任意页面
-> 右下角看到小助手按钮
-> 点击展开
-> 看到欢迎语和快捷问题
-> 输入问题或点击快捷问题
-> 等待小助手回答
-> 查看 action cards
-> 点击 action 跳转页面
-> 面板可保持打开或自动收起
```

## 29. 加载状态

发送消息后：

```text
输入框禁用
显示思考动画
防止重复发送
失败可以重试
```

失败文案：

```text
向导暂时没有响应，你可以稍后再试。
```

不要展示后端异常堆栈。

---

## 30. 视觉风格

必须延续 CodeCoach AI 现有风格：

```text

克制阴影
柔和动画
不做传统客服蓝色气泡风
```

小助手应该像产品的一部分，而不是第三方客服插件。

## 34. 隐私边界

小助手可以说：

```text
你最近 Redis 表现偏弱
你有 2 份文档还没有分析
你最近的复盘建议关注项目表达
```

不应该说：

```text
完整简历原文
完整手机号
完整训练回答
完整用户文档内容
```

## 35. 错误处理

常见错误：

```text
AI 服务失败
用户未登录
上下文数据读取失败
action 映射失败
网络异常
```

处理原则：

```text
用户可继续使用页面
小助手显示友好错误
不影响主业务
不阻塞导航
不暴露内部异常
```

---

## 36. MVP 范围

第一版做：

后端：

```text
/api/guide/chat
当前路径识别
用户数据摘要聚合
规则优先回答
可选 AI 个性化回答
action 白名单映射
权限控制
```

前端：

```text
右下角悬浮按钮
小助手面板
对话消息
快捷问题
行动卡片
点击跳转
加载与错误状态
全站接入
```

第一版不做：

```text
长期记忆
复杂 Agent 编排
自动创建训练
自动修改数据
WebSocket
语音输入
外部搜索
多 Agent 协作
```

## 37. 阶段完成标准

完成后应满足：

```text
用户在任意页面能打开小助手
小助手能解释当前页面
小助手能回答“下一步该做什么”
小助手能返回可点击 action
未登录时能引导登录
已登录时能给个性化建议
点击 action 能正确跳转
action 不会跳到非法路径
小助手失败不影响主业务
视觉风格融入当前产品
```

## 38. 简历表达方向

模块完成后可以描述为：

```text
设计并实现产品内 AI 训练向导，基于当前路由、用户训练状态、能力画像、复盘结果和简历风险点，为用户生成下一步训练建议，并通过 action card 引导用户快速进入知识学习、八股训练、项目拷打、简历训练和复盘 Agent 等功能。
```

进一步可以描述为：

```text
该向导采用规则优先与 AI 个性化建议结合的方式，通过白名单 action 映射保证跳转可控，避免通用聊天机器人不可控问题，实现了 AI 产品内部的智能导航和训练路径引导。
```

## 39. 总结

产品内 AI 训练向导是 CodeCoach AI 从“功能完整”走向“体验完整”的关键模块。

它不追求万能聊天，而是解决一个很实际的问题：

```text
用户现在应该做什么？
```

它将已有能力串联起来：

```text
项目训练
八股训练
知识学习
成长洞察
我的文档
简历训练
复盘 Agent
```

最终让产品更像一个真正的 AI 教练系统，而不是功能页面集合。
