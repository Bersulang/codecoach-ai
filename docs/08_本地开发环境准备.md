# CodeCoach AI 本地开发环境准备

## 1. 文档说明

本文档用于记录 CodeCoach AI 项目的本地开发环境要求、环境检查命令和常见问题。

CodeCoach AI 采用前后端分离架构：

```text
React 前端
    |
Spring Boot 后端
    |
MySQL / Redis
    |
外部大模型 API
```

MVP 阶段需要准备 Java、Maven、Node.js、Git、Docker 等基础开发工具。

---

## 2. 必要开发工具

| 工具 | 建议版本 | 用途 |
|---|---|---|
| JDK | 21 | 后端 Java 开发 |
| Maven | 3.9.x | 后端项目构建 |
| Node.js | 20 LTS | 前端开发 |
| npm / pnpm | npm 10+ / pnpm 9+ | 前端包管理 |
| Git | 2.x | 版本控制 |
| Docker Desktop | 最新稳定版 | 启动 MySQL、Redis 等中间件 |
| IntelliJ IDEA | 2024+ / 2025+ | 后端开发 |
| VS Code / WebStorm | 最新版 | 前端开发 |
| APIfox / Postman | 任意 | 接口测试 |

## 3. Mac 环境检查

## 3.1 检查 Java

```bash
java -version
```

期望版本：

```text

```

如果本机存在多个 JDK，需要确认 `JAVA_HOME`：

```bash
echo $JAVA_HOME
```

## 4. Windows 环境准备

Windows 开发时也需要准备：

1. JDK 21；
2. Maven 3.9.x；
3. Node.js 20 LTS；
4. Git；
5. Docker Desktop；
6. IntelliJ IDEA；
7. VS Code / WebStorm。

Windows 上需要特别注意：

1. Maven 使用的 JDK 版本是否正确；
2. Node.js 是否加入 PATH；
3. npm 执行策略问题；
4. Git 换行符配置；
5. Docker Desktop 是否正常启动。

---

## 5. Git 换行符配置

为了避免 Mac 和 Windows 协同开发时出现大量无意义换行变更，建议配置 Git 换行符。

Mac 推荐：

```bash
git config --global core.autocrlf input
```

Windows 推荐：

```bash
git config --global core.autocrlf true
```

也可以在项目中增加 `.gitattributes` 文件，统一换行规则。

后续项目可以添加：

```text
* text=auto
*.java text eol=lf
*.ts text eol=lf
*.tsx text eol=lf
*.js text eol=lf
*.jsx text eol=lf
*.md text eol=lf
*.yml text eol=lf
*.yaml text eol=lf
*.sql text eol=lf
```

## 7. 本地环境变量规划

后端敏感配置不要提交到 Git。

建议使用：

```text
application-local.yml
```

用于存放：

1. 数据库密码；
2. JWT 密钥；
3. 大模型 API Key；
4. 第三方服务地址。

前端敏感配置不要提交到 Git。

建议使用：

```text
.env.local
```

用于存放：

```text
VITE_API_BASE_URL=http://localhost:8080
```

项目中可以提交模板文件：

```text
application-example.yml
.env.example
```

## 8. 本地数据库规划

MVP 阶段数据库名：

```text
codecoach_ai
```

MySQL 用户可以先使用本地 root 用户开发，后续部署时创建独立用户。

示例：

```sql
CREATE DATABASE codecoach_ai DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

后续会在 `deploy` 目录中创建：

```text
deploy
└── docker-compose.yml
```

MVP 本地开发至少包含：

1. MySQL 8；
2. Redis 7。

后续部署阶段再增加：

1. 后端服务；
2. 前端 Nginx；
3. Nginx 反向代理配置。

## 10. 开发启动流程，规划版

后续项目初始化完成后，本地开发启动流程如下：

```bash
# 1. 拉取最新代码
git pull

# 2. 启动中间件
cd deploy
docker compose up -d

# 3. 启动后端
cd ../backend
mvn spring-boot:run

# 4. 启动前端
cd ../frontend
npm install
npm run dev
```
