# CodeCoach AI 页面流程设计

## 1. 文档说明

本文档用于描述 CodeCoach AI MVP 版本的页面结构、页面跳转关系、页面功能点和用户操作流程。

MVP 版本核心目标是完成“项目拷打训练闭环”：

用户登录 -> 创建项目档案 -> 发起项目拷打训练 -> 回答 AI 问题 -> 结束训练 -> 查看训练报告。

---

## 2. 页面清单

MVP 版本包含以下页面：

| 页面名称 | 路由 | 是否需要登录 | 说明 |
|---|---|---|---|
| 登录页 | `/login` | 否 | 用户登录系统 |
| 注册页 | `/register` | 否 | 用户注册账号 |
| 项目列表页 | `/projects` | 是 | 展示当前用户创建的项目档案 |
| 项目创建页 | `/projects/new` | 是 | 创建新的项目档案 |
| 项目编辑页 | `/projects/:id/edit` | 是 | 修改已有项目档案 |
| 项目拷打训练页 | `/interviews/:sessionId` | 是 | 用户与 AI 进行项目面试训练 |
| 训练报告页 | `/reports/:reportId` | 是 | 查看一次训练反馈报告 |
| 训练历史页 | `/history` | 是 | 查看历史训练记录 |

---

## 4. 页面详情设计

## 4.1 登录页

### 4.1.1 页面路径

```text
/login
```

### 4.1.2 页面目标

用户通过用户名和密码登录系统。

### 4.1.3 页面元素

| 元素 | 类型 | 说明 |
|---|---|---|
| 用户名输入框 | Input | 输入用户名 |
| 密码输入框 | Password Input | 输入密码 |
| 登录按钮 | Button | 提交登录 |
| 注册入口 | Link | 跳转注册页 |
| 错误提示 | Message | 展示登录失败原因 |

### 4.1.4 用户操作

#### 操作一：登录

用户输入用户名和密码，点击登录按钮。

前端校验：

1. 用户名不能为空；
2. 密码不能为空。

请求接口：

```http
POST /api/auth/login
```

请求参数：

```json
{
  "username": "zhangsan",
  "password": "123456"
}
```

成功后：

1. 保存 Token；
2. 保存用户基本信息；
3. 跳转到项目列表页 `/projects`。

失败后：

1. 展示错误提示；
2. 停留在登录页。

### 4.1.5 验收标准

1. 用户名和密码为空时，前端提示；
2. 用户名密码正确时，登录成功；
3. 登录成功后跳转项目列表页；
4. 登录失败时展示错误信息；
5. 已登录用户访问 `/login` 时，可自动跳转 `/projects`。

---

## 4.2 注册页

### 4.2.1 页面路径

```text
/register
```

### 4.2.2 页面目标

用户注册一个新账号。

### 4.2.3 页面元素

| 元素 | 类型 | 说明 |
|---|---|---|
| 用户名输入框 | Input | 输入用户名 |
| 密码输入框 | Password Input | 输入密码 |
| 确认密码输入框 | Password Input | 再次输入密码 |
| 注册按钮 | Button | 提交注册 |
| 登录入口 | Link | 跳转登录页 |
| 错误提示 | Message | 展示注册失败原因 |

### 4.2.4 用户操作

#### 操作一：注册

用户填写用户名、密码和确认密码，点击注册。

前端校验：

1. 用户名不能为空；
2. 用户名长度为 4-20；
3. 密码不能为空；
4. 密码长度为 6-32；
5. 两次密码必须一致。

请求接口：

```http
POST /api/auth/register
```

请求参数：

```json
{
  "username": "zhangsan",
  "password": "123456",
  "confirmPassword": "123456"
}
```

成功后：

1. 展示注册成功提示；
2. 跳转登录页 `/login`。

失败后：

1. 展示错误提示；
2. 停留在注册页。

### 4.2.5 验收标准

1. 合法信息可以注册成功；
2. 用户名重复时提示注册失败；
3. 两次密码不一致时前端提示；
4. 注册成功后跳转登录页。

---

## 4.3 项目列表页

### 4.3.1 页面路径

```text
/projects
```

### 4.3.2 页面目标

展示当前用户创建的项目档案，并提供新建、编辑、删除、开始训练等操作。

### 4.3.3 页面元素

| 元素 | 类型 | 说明 |
|---|---|---|
| 页面标题 | Text | 我的项目 |
| 新建项目按钮 | Button | 跳转项目创建页 |
| 搜索框 | Input | 按项目名称搜索 |
| 项目列表 | Table / Card List | 展示项目档案 |
| 编辑按钮 | Button | 编辑项目 |
| 删除按钮 | Button | 删除项目 |
| 开始拷打按钮 | Button | 基于项目创建训练 |
| 历史记录入口 | Link / Button | 跳转训练历史页 |

### 4.3.4 项目列表字段

| 字段 | 说明 |
|---|---|
| 项目名称 | 项目名称 |
| 技术栈 | 项目使用的核心技术 |
| 负责模块 | 用户负责的模块 |
| 创建时间 | 项目创建时间 |
| 更新时间 | 项目最后更新时间 |
| 操作 | 编辑 / 删除 / 开始拷打 |

### 4.3.5 用户操作

#### 操作一：进入页面加载项目列表

请求接口：

```http
GET /api/projects?pageNum=1&pageSize=10
```

成功后：

1. 渲染项目列表；
2. 如果无数据，展示空状态。

#### 操作二：搜索项目

用户输入关键词，点击搜索。

请求接口：

```http
GET /api/projects?keyword=营销&pageNum=1&pageSize=10
```

成功后：

1. 展示匹配结果；
2. 无匹配结果时展示空状态。

#### 操作三：新建项目

点击“新建项目”。

跳转：

```text
/projects/new
```

#### 操作四：编辑项目

点击某个项目的“编辑”。

跳转：

```text
/projects/:id/edit
```

#### 操作五：删除项目

点击“删除”。

前端行为：

1. 弹出二次确认；
2. 用户确认后请求删除接口。

请求接口：

```http
DELETE /api/projects/:id
```

成功后：

1. 展示删除成功；
2. 重新加载项目列表。

#### 操作六：开始拷打

点击“开始拷打”。

请求接口：

```http
POST /api/interview-sessions
```

请求参数：

```json
{
  "projectId": 1,
  "targetRole": "Java 后端实习",
  "difficulty": "NORMAL"
}
```

成功后：

1. 获得训练会话 ID；
2. 跳转项目拷打训练页。

跳转：

```text
/interviews/:sessionId
```

### 4.3.6 验收标准

1. 用户进入页面后可以看到自己的项目列表；
2. 无项目时展示空状态；
3. 搜索功能可用；
4. 新建、编辑、删除入口可用；
5. 点击开始拷打后可以创建训练会话；
6. 用户不能看到其他用户的项目。

## 4.4 项目创建页

### 4.4.1 页面路径

```text
/projects/new
```

### 4.4.2 页面目标

用户填写项目信息，创建项目档案。

### 4.4.3 页面元素

| 元素 | 类型 | 说明 |
|---|---|---|
| 项目名称 | Input | 必填 |
| 项目描述 | TextArea | 必填 |
| 技术栈 | TextArea / Select Tags | 必填 |
| 负责模块 | TextArea | 可选 |
| 项目亮点 | TextArea | 可选 |
| 项目难点 | TextArea | 可选 |
| 保存按钮 | Button | 保存项目 |
| 取消按钮 | Button | 返回项目列表 |

### 4.4.4 表单字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| name | string | 是 | 项目名称 |
| description | string | 是 | 项目描述 |
| techStack | string | 是 | 技术栈 |
| role | string | 否 | 负责模块 |
| highlights | string | 否 | 项目亮点 |
| difficulties | string | 否 | 项目难点 |

### 4.4.5 用户操作

#### 操作一：保存项目

前端校验：

1. 项目名称不能为空；
2. 项目描述不能为空；
3. 技术栈不能为空。

请求接口：

```http
POST /api/projects
```

请求参数：

```json
{
  "name": "高并发营销系统",
  "description": "该项目面向营销活动场景，支持优惠券发放、抽奖、秒杀等能力。",
  "techStack": "Spring Boot, MySQL, Redis, RabbitMQ",
  "role": "负责优惠券秒杀模块和库存扣减逻辑",
  "highlights": "使用 Redis 预扣库存，结合 MQ 异步削峰。",
  "difficulties": "需要处理超卖、重复消费、缓存一致性等问题。"
}
```

成功后：

1. 展示保存成功；
2. 跳转项目列表页。

失败后：

1. 展示错误信息；
2. 停留在当前页。

#### 操作二：取消

点击取消按钮。

跳转：

```text
/projects
```

### 4.4.6 验收标准

1. 必填字段为空时不能提交；
2. 合法信息可以创建项目；
3. 创建成功后跳转项目列表页；
4. 创建失败时展示错误信息。

4. 修改其他用户的项目失败。

## 4.6 项目拷打训练页

### 4.6.1 页面路径

```text
/interviews/:sessionId
```

### 4.6.2 页面目标

用户基于某个项目与 AI 进行多轮项目面试训练。

### 4.6.3 页面元素

| 元素 | 类型 | 说明 |
|---|---|---|
| 项目信息区域 | Card | 展示项目名称、技术栈、目标岗位、难度 |
| 对话区域 | Chat List | 展示 AI 问题、用户回答、AI 反馈 |
| 回答输入框 | TextArea | 用户输入回答 |
| 提交回答按钮 | Button | 提交当前回答 |
| 结束训练按钮 | Button | 结束当前训练 |
| 当前轮次提示 | Text | 例如第 2 / 5 轮 |
| Loading 状态 | Spin | AI 生成中提示 |

### 4.6.4 对话消息类型

| 类型 | 说明 |
|---|---|
| AI_QUESTION | AI 面试问题 |
| USER_ANSWER | 用户回答 |
| AI_FEEDBACK | AI 对本轮回答的反馈 |
| AI_FOLLOW_UP | AI 继续追问 |

### 4.6.5 用户操作

#### 操作一：进入页面加载训练会话

请求接口：

```http
GET /api/interview-sessions/:sessionId
```

成功后：

1. 展示项目信息；
2. 展示已有对话记录；
3. 如果是新会话，展示第一道 AI 问题。

#### 操作二：提交回答

前端校验：

1. 回答内容不能为空；
2. AI 正在生成时不能重复提交；
3. 已结束会话不能提交。

请求接口：

```http
POST /api/interview-sessions/:sessionId/answer
```

请求参数：

```json
{
  "questionId": 1,
  "answer": "这个项目主要面向营销活动场景，我负责优惠券秒杀模块..."
}
```

成功后：

1. 对话区追加用户回答；
2. 展示 AI 本轮反馈；
3. 展示 AI 下一轮追问；
4. 清空输入框。

失败后：

1. 展示错误信息；
2. 保留用户输入内容。

#### 操作三：结束训练

点击“结束训练”。

前端行为：

1. 弹出二次确认；
2. 用户确认后请求结束接口。

请求接口：

```http
POST /api/interview-sessions/:sessionId/finish
```

成功后：

1. 生成训练报告；
2. 跳转训练报告页。

跳转：

```text
/reports/:reportId
```

### 4.6.6 验收标准

1. 进入页面后可以看到当前训练会话；
2. 可以看到第一道 AI 面试题；
3. 用户提交回答后可以得到 AI 反馈和追问；
4. 回答为空时不能提交；
5. AI 生成中不能重复提交；
6. 用户可以结束训练并进入报告页；
7. 已结束训练不能继续回答。

## 4.7 训练报告页

```text
/reports/:reportId
```

| suggestions | 改进建议 |

### 4.7.5 用户操作

#### 操作一：进入页面加载报告

```http
GET /api/reports/:reportId
```

```text
/projects
```

请求参数：

```json
{
  "projectId": 1,
  "targetRole": "Java 后端实习",
  "difficulty": "NORMAL"
}
```

成功后：

1. 跳转新的训练会话页面。

### 4.7.6 验收标准

1. 用户可以查看自己的训练报告；

3. 用户不能查看其他用户的报告；
4. 返回项目列表功能可用；
5. 再来一次功能可用。

### 4.8.6 验收标准

1. 用户可以看到自己的历史训练记录；
2. 可以按项目筛选；
3. 已完成训练可以查看报告；
4. 未完成训练可以继续训练；
5. 用户不能看到其他用户的训练记录。

## 5. 登录态与路由权限

### 5.1 无需登录页面

以下页面无需登录：

```text
/login
/register
```

### 5.2 需要登录页面

以下页面需要登录：

```text
/projects
/projects/new
/projects/:id/edit
/interviews/:sessionId
/reports/:reportId
/history
```

### 5.3 路由拦截规则

1. 未登录用户访问受保护页面时，跳转 `/login`；
2. 已登录用户访问 `/login` 或 `/register` 时，跳转 `/projects`；
3. Token 过期时，清除本地登录信息并跳转 `/login`；
4. 用户无权限访问资源时，展示错误提示并返回项目列表页。

## 6. 页面状态设计

### 6.1 Loading 状态

以下场景需要展示 Loading：

1. 登录提交中；
2. 注册提交中；
3. 项目列表加载中；
4. 项目保存中；
5. AI 生成问题中；
6. AI 生成反馈中；
7. 报告生成中；
8. 历史记录加载中。

### 6.2 Empty 状态

以下场景需要展示 Empty：

1. 用户没有创建任何项目；
2. 搜索不到项目；
3. 用户没有历史训练记录；
4. 报告问答复盘为空。

### 6.3 Error 状态

以下场景需要展示错误提示：

1. 登录失败；
2. 注册失败；
3. 项目保存失败；
4. 项目加载失败；
5. AI 调用失败；
6. 报告加载失败；
7. 无权限访问资源。

## 8. MVP 页面完成标准

当以下页面流程可以完整跑通时，页面层面 MVP 完成：

1. 用户可以从注册页注册账号；
2. 用户可以从登录页登录系统；
3. 登录后进入项目列表页；
4. 用户可以创建项目档案；
5. 创建成功后项目出现在项目列表；
6. 用户可以基于项目开始拷打训练；
7. 用户可以在训练页回答 AI 问题；
8. 用户可以结束训练；
9. 用户可以查看训练报告；
10. 用户退出后无法访问受保护页面。
