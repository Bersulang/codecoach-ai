# CodeCoach AI 需求规格说明书

## 1. 文档说明

本文档用于描述 CodeCoach AI MVP 版本的功能需求、用户角色、业务流程、功能优先级和验收标准。

MVP 版本目标是完成“项目拷打训练闭环”：

用户创建项目档案 -> 发起项目拷打训练 -> AI 生成面试追问 -> 用户回答 -> AI 继续追问 -> 用户结束训练 -> AI 生成训练反馈。

---

## 2. 用户角色

### 2.1 游客

游客是未登录用户。

MVP 阶段游客暂不支持核心功能，仅可以访问首页和登录 / 注册页面。

后续版本可以支持游客体验示例项目拷打。

### 2.2 注册用户

注册用户是 MVP 阶段的核心用户。

注册用户可以：

1. 注册账号；
2. 登录系统；
3. 创建项目档案；
4. 查看项目列表；
5. 编辑项目档案；
6. 删除项目档案；
7. 基于某个项目发起项目拷打训练；
8. 在训练过程中回答 AI 面试问题；
9. 结束训练并查看反馈报告；
10. 查看历史训练记录。

### 2.3 管理员

MVP 阶段暂不实现管理员功能。

管理员功能可在后续版本中扩展，例如用户管理、Prompt 模板管理、模型调用日志查看等。

---

## 3. MVP 功能范围

MVP 版本只实现以下核心模块：

1. 用户认证模块；
2. 项目档案模块；
3. 项目拷打训练模块；
4. 训练反馈报告模块；
5. 历史记录模块。

暂不实现以下功能：

1. 文档上传；
2. PDF / Word / Markdown 解析；
3. RAG 知识库；
4. 向量数据库；
5. 多 Agent 编排；
6. 语音面试；
7. 视频面试；
8. 支付系统；
9. 管理员后台；
10. 企业组织管理。

---

## 4. 核心业务流程

### 4.1 用户注册登录流程

1. 用户进入注册页面；
2. 输入用户名、密码、确认密码；
3. 系统校验用户名是否重复；
4. 注册成功后跳转登录页面；
5. 用户输入用户名和密码；
6. 登录成功后系统返回 Token；
7. 前端保存 Token；
8. 用户进入项目列表页面。

### 4.2 创建项目档案流程

1. 用户进入项目列表页面；
2. 点击“新建项目”；
3. 输入项目名称、项目描述、技术栈、负责模块、项目亮点、项目难点；
4. 点击保存；
5. 系统保存项目档案；
6. 用户回到项目列表页面。

### 4.3 项目拷打训练流程

1. 用户在项目列表中选择一个项目；
2. 点击“开始拷打”；
3. 系统创建一次训练会话；
4. AI 根据项目档案生成第一个面试问题；
5. 用户输入回答；
6. 系统将项目档案、历史问答和用户回答发送给 AI；
7. AI 生成反馈和下一轮追问；
8. 用户继续回答；
9. 用户点击“结束训练”；
10. 系统生成训练反馈报告。

### 4.4 查看训练报告流程

1. 用户进入历史记录页面；
2. 选择一次训练记录；
3. 查看本次训练的问题、回答、AI 反馈、总评分、薄弱点和改进建议。

## 5. 功能需求详情

## 5.1 用户认证模块

### 5.1.1 用户注册

#### 功能描述

用户可以通过用户名和密码注册账号。

#### 输入字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| username | string | 是 | 用户名，长度 4-20 |
| password | string | 是 | 密码，长度 6-32 |
| confirmPassword | string | 是 | 确认密码 |

#### 业务规则

1. 用户名不能为空；
2. 用户名不能重复；
3. 密码不能为空；
4. 密码和确认密码必须一致；
5. 注册成功后默认角色为 USER。

#### 验收标准

1. 输入合法信息时，注册成功；
2. 用户名重复时，注册失败并提示；
3. 两次密码不一致时，注册失败并提示；
4. 注册成功的数据可以在数据库中查询到。

---

### 5.1.2 用户登录

#### 功能描述

用户可以通过用户名和密码登录系统。

#### 输入字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| username | string | 是 | 用户名 |
| password | string | 是 | 密码 |

#### 业务规则

1. 用户名和密码不能为空；
2. 用户名不存在时，登录失败；
3. 密码错误时，登录失败；
4. 登录成功后返回 Token 和用户基本信息。

#### 验收标准

1. 输入正确用户名和密码时，登录成功；
2. 输入错误密码时，登录失败；
3. 登录成功后前端可以访问受保护接口；
4. 未登录用户访问受保护接口时返回 401。

---

## 5.2 项目档案模块

### 5.2.1 创建项目档案

#### 功能描述

用户可以创建自己的项目档案，用于后续 AI 面试训练。

#### 输入字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| name | string | 是 | 项目名称 |
| description | string | 是 | 项目描述 |
| techStack | string | 是 | 技术栈 |
| role | string | 否 | 用户负责模块 |
| highlights | string | 否 | 项目亮点 |
| difficulties | string | 否 | 项目难点 |

#### 业务规则

1. 项目名称不能为空；
2. 项目描述不能为空；
3. 技术栈不能为空；
4. 项目只能归属于当前登录用户；
5. 用户只能查看、修改、删除自己的项目。

#### 验收标准

1. 用户可以成功创建项目；
2. 创建后项目出现在项目列表中；
3. 用户 A 不能查看用户 B 的项目；
4. 必填字段为空时创建失败。

---

### 5.2.2 查询项目列表

#### 功能描述

用户可以查看自己创建的项目档案列表。

#### 查询条件

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| keyword | string | 否 | 按项目名称模糊搜索 |
| pageNum | number | 否 | 页码 |
| pageSize | number | 否 | 每页条数 |

#### 业务规则

1. 只返回当前登录用户的项目；
2. 默认按创建时间倒序排序；
3. 支持分页查询。

#### 验收标准

1. 登录用户可以看到自己的项目列表；
2. 搜索关键词时只返回匹配项目；
3. 分页参数生效；
4. 不会返回其他用户的项目。

---

### 5.2.3 修改项目档案

#### 功能描述

用户可以修改自己的项目档案。

#### 业务规则

1. 只能修改自己的项目；
2. 项目名称、项目描述、技术栈不能为空；
3. 修改后更新时间需要变化。

#### 验收标准

1. 修改自己的项目成功；
2. 修改不存在的项目失败；
3. 修改其他用户的项目失败。

---

### 5.2.4 删除项目档案

#### 功能描述

用户可以删除自己的项目档案。

#### 业务规则

1. 只能删除自己的项目；
2. 删除项目时，暂不物理删除关联训练记录；
3. MVP 阶段可以采用逻辑删除。

#### 验收标准

1. 删除自己的项目成功；
2. 删除后项目列表不再显示；
3. 删除其他用户的项目失败。

---

## 5.3 项目拷打训练模块

### 5.3.1 创建训练会话

#### 功能描述

用户可以基于某个项目创建一次项目拷打训练会话。

#### 输入字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| projectId | number | 是 | 项目 ID |
| targetRole | string | 是 | 目标岗位，例如 Java 后端实习 |
| difficulty | string | 是 | 难度：EASY / NORMAL / HARD |

#### 业务规则

1. 项目必须属于当前用户；
2. 创建会话后状态为 IN_PROGRESS；
3. 系统需要生成第一道面试问题；
4. 训练会话需要记录开始时间。

#### 验收标准

1. 选择自己的项目可以成功创建训练会话；
2. 创建后可以获得第一道 AI 面试问题；
3. 使用其他用户的项目创建训练会话失败。

---

### 5.3.2 回答面试问题

#### 功能描述

用户在训练会话中回答 AI 提出的问题，系统根据用户回答生成反馈和下一轮追问。

#### 输入字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| sessionId | number | 是 | 训练会话 ID |
| questionId | number | 是 | 当前问题 ID |
| answer | string | 是 | 用户回答 |

#### 业务规则

1. 会话必须属于当前用户；
2. 会话状态必须为 IN_PROGRESS；
3. 回答不能为空；
4. 系统保存用户回答；
5. AI 生成本轮反馈和下一道追问；
6. 每次训练最多进行 5 轮，MVP 阶段可固定轮数。

#### 验收标准

1. 用户提交回答后保存成功；
2. 系统返回本轮反馈；
3. 系统返回下一道追问；
4. 回答空内容时提交失败；
5. 已结束会话不能继续回答。

---

### 5.3.3 结束训练会话

#### 功能描述

用户可以主动结束训练会话，系统生成训练反馈报告。

#### 业务规则

1. 会话必须属于当前用户；
2. 会话结束后状态变为 FINISHED；
3. 系统记录结束时间；
4. 系统基于本次所有问答生成训练报告。

#### 验收标准

1. 用户可以结束自己的训练会话；
2. 结束后生成训练报告；
3. 结束后的会话不能继续回答；
4. 用户不能结束其他用户的会话。

---

## 5.4 训练反馈报告模块

### 5.4.1 查看训练报告

#### 功能描述

用户可以查看一次训练会话对应的反馈报告。

#### 报告内容

| 字段 | 说明 |
|---|---|
| totalScore | 总评分 |
| summary | 总体评价 |
| strengths | 优点 |
| weaknesses | 薄弱点 |
| suggestions | 改进建议 |
| qaReview | 问答复盘 |

#### 业务规则

1. 报告必须归属于当前用户；
2. 训练结束后才能生成报告；
3. 用户只能查看自己的报告。

#### 验收标准

1. 用户可以查看自己的训练报告；
2. 未结束训练时不能查看报告；
3. 用户不能查看其他用户的报告。

---

## 5.5 历史记录模块

### 5.5.1 查看训练历史

#### 功能描述

用户可以查看自己的历史训练记录。

#### 查询条件

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| projectId | number | 否 | 按项目筛选 |
| status | string | 否 | 按状态筛选 |
| pageNum | number | 否 | 页码 |
| pageSize | number | 否 | 每页条数 |

#### 业务规则

1. 只返回当前用户的训练记录；
2. 默认按创建时间倒序排序；
3. 支持分页。

#### 验收标准

1. 用户可以查看自己的训练历史；
2. 支持按项目筛选；
3. 不会返回其他用户的训练记录。

## 6. 功能优先级

### P0：MVP 必须完成

1. 用户注册；
2. 用户登录；
3. 获取当前用户信息；
4. 创建项目档案；
5. 查询项目列表；
6. 创建训练会话；
7. 回答面试问题；
8. 结束训练并生成报告；
9. 查看训练报告。

### P1：MVP 尽量完成

1. 修改项目档案；
2. 删除项目档案；
3. 查看训练历史；
4. 项目关键词搜索；
5. 训练轮数限制；
6. 模型调用日志记录。

### P2：后续版本

1. 文档上传；
2. RAG 知识库；
3. 向量数据库；
4. SSE 流式输出；
5. 游客体验模式；
6. Prompt 模板管理；
7. 管理员后台；
8. 部署监控。

## 7. 关键数据状态

### 7.1 项目状态

MVP 阶段项目默认只有正常和删除两种状态：

| 状态 | 说明 |
|---|---|
| NORMAL | 正常 |
| DELETED | 已删除 |

### 7.2 训练会话状态

| 状态 | 说明 |
|---|---|
| IN_PROGRESS | 训练中 |
| FINISHED | 已结束 |
| FAILED | 生成失败 |

### 7.3 难度等级

| 状态 | 说明 |
|---|---|
| EASY | 简单 |
| NORMAL | 普通 |
| HARD | 困难 |

## 8. 非功能需求

### 8.1 安全性

1. 密码不能明文存储；
2. 受保护接口必须校验 Token；
3. 用户只能访问自己的数据；
4. 大模型 API Key 不能提交到 Git；
5. 生产环境配置不能提交到 Git。

### 8.2 可用性

1. AI 调用失败时需要给出友好提示；
2. 页面加载时需要展示 loading；
3. 表单提交失败时需要展示错误信息；
4. 用户误操作时需要二次确认，例如删除项目。

### 8.3 可维护性

1. 后端统一响应格式；
2. 后端统一异常处理；
3. 业务错误使用自定义异常；
4. 接口命名保持 REST 风格；
5. 前端 API 请求统一封装。

### 8.4 可测试性

1. 每个核心接口应支持通过 API 工具测试；
2. 每个核心业务流程应有明确验收标准；
3. AI 输出不要求完全固定，但结构必须稳定。

---

## 9. MVP 完成标准

当以下流程完整可用时，MVP 视为完成：

1. 用户注册账号；
2. 用户登录系统；
3. 用户创建项目档案；
4. 用户基于项目发起项目拷打训练；
5. 系统生成第一道项目面试题；
6. 用户回答问题；
7. 系统生成反馈和追问；
8. 用户结束训练；
9. 系统生成训练报告；
10. 用户可以查看历史训练报告。
