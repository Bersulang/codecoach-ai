# Spring AOP

## 1. 一句话解释

Spring AOP 是通过动态代理把日志、事务、权限等横切逻辑从业务代码中抽离出来的一种机制。

## 2. 面试标准回答

面试时可以先说 AOP 解决什么问题：很多逻辑不是业务本身，但会散落在多个方法里，比如日志记录、权限校验、事务控制、接口耗时统计。AOP 可以把这些横切逻辑集中管理，减少重复代码。

Spring AOP 的核心是代理。对于实现接口的类，默认可以使用 JDK 动态代理；对于没有接口的类，可以使用 CGLIB 生成子类代理。业务方法调用代理对象时，代理对象在目标方法前后织入通知逻辑。

常见概念包括切点、通知、切面和连接点。切点定义哪些方法被增强，通知定义增强逻辑在方法前、后、异常时还是环绕执行。Spring 事务本质上也依赖 AOP，通过代理在方法执行前开启事务，执行后提交或回滚。

## 3. 核心原理

Spring AOP 是运行时代理，不是直接修改字节码文件。容器创建 Bean 时，如果匹配切面规则，会生成代理对象放入容器。外部注入和调用的是代理对象，代理对象再调用目标对象。

这也解释了自调用失效问题：同一个类内部 `this.method()` 调用不会经过代理对象，所以事务、权限等 AOP 增强不会生效。private 方法、final 方法等也可能因为代理机制无法正常增强。

JDK 动态代理基于接口，CGLIB 基于继承。选择时要说明差异，而不是简单说谁更好。

## 5. 易错点

不要只说“AOP 是面向切面编程”，这太抽象。要讲清它解决重复横切逻辑的问题。不要忽略代理对象这个核心，否则解释不了事务失效。

回答事务时不要把 `@Transactional` 当成魔法。它依赖 Spring AOP，必须通过代理对象调用，异常类型和传播行为也会影响事务结果。

## 6. 项目表达模板

在项目中，我会把接口耗时统计、操作日志、权限校验这类横切逻辑放到 AOP 中处理。比如通过自定义注解标记需要记录操作日志的方法，切面中获取方法参数、用户信息和执行结果统一落库。这样业务方法只关注业务逻辑，日志规则变化也不用改多个 Service。

## 7. 推荐训练

建议继续训练动态代理、Bean 生命周期、事务传播和 Spring Boot 自动配置。AOP 题要重点练习“代理机制 -> 自调用失效 -> 项目应用场景”。
